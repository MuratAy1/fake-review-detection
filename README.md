A machine learning pipeline that detects fake (computer-generated) product reviews using NLP preprocessing and TF-IDF feature extraction. Four classifiers are compared (Naive Bayes, Logistic Regression, SVM, Random Forest), the final model is deployed as a Streamlit web app with an abstention mechanism for low-confidence predictions, and an external test set is used to evaluate generalization beyond the training distribution.

## Project Overview

### Problem

Online fake reviews mislead consumers and distort e-commerce markets. Manual detection is impractical at scale, and human reviewers detect fakes at near-chance level (Ott et al., 2011). Automated ML-based detection is essential.

### Approach

```
Raw Reviews -> NLP Preprocessing -> TF-IDF Vectorization -> ML Classification -> Evaluation
                                                                                  |
                                                                                  v
External Test Set ----> Robustness Analysis -----> Abstention Mechanism ----> Streamlit App
```

### Key Findings

- **SVM outperforms** all other classical models on the internal test, consistent with literature (Joachims, 1998; Ott et al., 2011)
- **Random Forest underperforms** on sparse TF-IDF matrices because decision trees split on individual features, while text information is distributed across thousands of dimensions
- **5-fold cross-validation** confirms no overfitting; SVM has the lowest variance
- **External test reveals generator shift**: a model trained on GPT-2 era data does not generalize to fake reviews produced by modern LLMs (2/50 detection)
- **Abstention mechanism** mitigates the practical risk: low-confidence predictions are labeled "UNCERTAIN" instead of forcing a binary decision

## Dataset

**[Kaggle Fake Reviews Dataset](https://www.kaggle.com/datasets/mexwell/fake-reviews-dataset)** (Salminen et al., 2022) — 40,432 product reviews:

- 20,216 **genuine** (human-written, labeled `OR`)
- 20,216 **fake** (GPT-2 generated, labeled `CG`)
- Categories: Home & Kitchen, Clothing, Electronics, Sports & Outdoors, etc.
- Perfectly balanced — no class imbalance handling required

> Note: The fake reviews were generated by GPT-2, which was released by OpenAI in 2019. This is relevant when evaluating generalization to fake reviews from newer language models.

## Pipeline Details

### 1. Text Preprocessing

- Lowercasing
- URL and HTML tag removal (regex)
- Number and punctuation removal
- Stop-word removal (NLTK English stopwords)
- Lemmatization (WordNet Lemmatizer)

### 2. Feature Extraction (TF-IDF)

| Parameter | Value | Rationale |
| --- | --- | --- |
| `max_features` | 20,000 | Information / speed balance |
| `ngram_range` | (1, 2) | Unigrams + bigrams to capture phrases like "not good" |
| `max_df` | 0.95 | Drop terms appearing in >95% of documents |
| `min_df` | 2 | Drop terms appearing in fewer than 2 documents |
| `sublinear_tf` | True | Logarithmic TF scaling to dampen high-frequency terms |

### 3. Train / Test Split

- 80% training (32,344 reviews) / 20% test (8,087 reviews)
- Stratified sampling to preserve class balance
- TF-IDF fitted on training set only (no data leakage)

### 4. Models & Hyperparameters

| Model | Key Parameters |
| --- | --- |
| MultinomialNB | `alpha=0.5` |
| LogisticRegression | `C=1.0`, `max_iter=5000`, `class_weight=balanced`, `solver=liblinear` |
| LinearSVC | `C=1.0`, `max_iter=5000`, `class_weight=balanced` |
| RandomForest | `n_estimators=200`, `class_weight=balanced`, `n_jobs=-1` |

### 5. Robustness Analysis

A 100-sample external test set (50 OR + 50 CG, sourced outside Kaggle) is used to evaluate real-world generalization. Three mitigation strategies were tested:

- **Threshold tuning** — reduced false negatives but at unacceptable false positive cost
- **Data augmentation** — improved internal score by 0.0024 but did not solve external generalization
- **DistilBERT fine-tuning** — reached 0.9737 internal accuracy but produced 5/50 external detection with >99% overconfidence (classic overfitting)

### 6. Abstention Mechanism

The Streamlit deployment uses a two-criterion abstention rule for low-confidence predictions:

- Criterion 1: probability-like confidence < 0.55
- Criterion 2: |raw SVM decision score| < 0.20

If either criterion is met, the system returns "UNCERTAIN" rather than a binary label. This is aligned with selective classification (El-Yaniv & Geifman, 2010).

## Demo (Streamlit Web App)

The final model is wrapped in a Streamlit interface that classifies a user-entered review in real time (<10 ms inference).

```bash
streamlit run app.py
```

The interface returns one of three outputs:
- **REAL** (high confidence, predicted as genuine)
- **FAKE** (high confidence, predicted as computer-generated)
- **UNCERTAIN** (low confidence — abstention triggered)
